# ZABNIX WEBSITE PLATFORM — IMPLEMENTATION PLAN

## 1. Development Methodology & Sequence

To build the ZABNIX website platform, we follow a strict **10-step sequence** designed to minimize architectural errors and prevent visual lag:

```
[1. Foundation] ---> [2. CMS Integration] ---> [3. Design Token System]
                                                        |
                                                        v
[6. Motion Polish] <-- [5. Internal Pages] <-- [4. Homepage Core Layout]
       |
       v
[7. SEO Implementation] ---> [8. Optimization] ---> [9. Testing] ---> [10. Deploy]
```

---

## 2. Phase-by-Phase Roadmap

### Phase 1: Project Foundation
- **Goal:** Instantiate core file assets and verify runtime tooling.
- **Tasks:**
  1. Initialize Next.js 15:
     ```bash
     npx create-next-app@latest ./ --typescript --tailwind --app --src-dir=false --eslint
     ```
  2. Install dependencies:
     ```bash
     npm install framer-motion gsap lenis next-sanity sanity clsx tailwind-merge lucide-react
     ```
  3. Validate folders: Ensure `app/`, `components/`, `lib/`, `hooks/`, `types/`, `styles/`, `public/` directories match layout structures.

---

### Phase 2: Sanity CMS Setup
- **Goal:** Connect local schemas to the cloud-managed content database.
- **Tasks:**
  1. Initialize Sanity schema definitions under a `sanity/` root directory.
  2. Embed Sanity Studio routing at `app/admin/[[...tool]]/page.tsx`.
  3. Configure Client files: Write `lib/sanity/client.ts`, custom query strings in `lib/sanity/queries.ts`, and helper functions in `lib/sanity/image.ts`.
  4. Bind environment variables (`.env.local`):
     ```env
     NEXT_PUBLIC_SANITY_PROJECT_ID="your_project_id"
     NEXT_PUBLIC_SANITY_DATASET="production"
     ```

---

### Phase 3: Global Style System
- **Goal:** Standardize layout variables, spacing rules, and font definitions.
- **Tasks:**
  1. Map font families (Geist/Inter/Satoshi) in `app/layout.tsx`.
  2. Define custom color schemes, border rules, and backdrop blurs inside `globals.css` and `tailwind.config.js`.
  3. Code reusable atomic items: `components/common/Button.tsx`, `components/common/Container.tsx`, and `components/common/GlowWrapper.tsx`.

---

### Phase 4: Homepage Development
- **Goal:** Code the primary landing layout containing key scroll sections.
- **Tasks:**
  1. Create a `components/sections/` index containing `Hero.tsx`, `Ecosystem.tsx`, `Services.tsx`, `Products.tsx`, and `ProjectsPreview.tsx`.
  2. Incorporate the dynamic page entry component (`components/common/Loader.tsx`) inside the home root view.
  3. Verify responsive bounds across desktop, tablet, and mobile layouts.

---

### Phase 5: Routing & Internal Pages
- **Goal:** Bind CMS queries to dynamic Next.js routes.
- **Tasks:**
  1. Code dynamic slug sub-folders: `/blog/[slug]`, `/projects/[slug]`, `/careers/[slug]`.
  2. Build listings engines: `/blog/page.tsx` (filtered lists), `/careers/page.tsx` (position boards with resume uploads), `/about/page.tsx` (historical overview and Sahakar backing details).
  3. Establish dynamic SEO header hooks on all route layouts.

---

### Phase 6: Scroll & Motion Systems
- **Goal:** Inject cinematic fluid transitions and scroll interpolation.
- **Tasks:**
  1. Integrate Lenis smooth scrolling by wrapping page components in a layout provider.
  2. Build scroll entrance animations using Framer Motion scroll triggers.
  3. Embed GSAP timeline overrides on the landing Hero container for structural line-draw canvas reveals.
  4. Enforce strict GC routines on animations to prevent memory leaks and frames-per-second (FPS) degradation.

---

### Phase 7: SEO & Indexing
- **Goal:** Optimize search engine crawlability and structure.
- **Tasks:**
  1. Bind dynamic metadata attributes to static templates.
  2. Implement an autogenerated `sitemap.xml` route.
  3. Add JSON-LD structured schema markers (`Organization`, `Article`, `JobPosting`).

---

### Phase 8: Performance Tuning & Optimization
- **Goal:** Achieve target Lighthouse scores of 90+.
- **Tasks:**
  1. Convert rasterized graphics to high-efficiency AVIF formats.
  2. Implement size overrides on images: `<Image src={...} placeholder="blur" />`.
  3. Apply lazy loading to non-critical heavy modules.
  4. Compile final production build to identify dependency bloat.

---

### Phase 9: Testing & Validation
- **Goal:** Verify functional and responsive stability.
- **Tasks:**
  1. Form validations (Honeypot tests, email syntax checks).
  2. Mobile viewport checks: verify that heavy animations degrade gracefully.
  3. Cross-browser tests (Safari rendering alignments, Firefox backdrop filter fallbacks).

---

### Phase 10: Production Deployment
- **Goal:** Deploy the platform to a live global production domain.
- **Tasks:**
  1. Connect GitHub repository to Vercel dashboard.
  2. Deploy production build of the Next.js app.
  3. Bind custom domain name.
  4. Verify production env variables and SSL bindings.
