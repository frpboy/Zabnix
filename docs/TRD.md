### 1.1 Architectural Model
The ZABNIX platform uses a modern decoupled architecture:
1. **Headless CMS Layer:** Sanity houses structural schemas and acts as the content lake. Editor operations are performed in an embedded Sanity Studio route.
2. **Frontend Layer:** A Next.js 15 (App Router) client pre-renders layouts, executes API requests using GROQ query selectors, and delivers optimized HTML, CSS, and JS components via Vercel's global edge network.

## 4. Rendering & Caching Strategy

The project employs a hybrid rendering approach to achieve optimal performance and immediate SEO accessibility:

### 4.1 Static Site Generation (SSG)
- **Applicable Routes:** `/`, `/about`, `/services`, `/contact`.
- **Method:** Pages are compiled during build-time. Global navigation headers and marketing text are embedded as static components.

### 4.2 Incremental Static Regeneration (ISR)
- **Applicable Routes:** `/blog`, `/projects`, `/careers`.
- **Method:** Revalidate setting is capped at `60` seconds (`export const revalidate = 60;`). 
- **Trigger:** When content is modified in Sanity, the system triggers Vercel revalidation. Visitors continue viewing cached HTML until background regenerations complete.

## 5. Animation & Scroll Framework

### 5.1 Scroll Interpolation (Lenis)
- Instantiated via a global React provider wrapping `app/layout.tsx`.
- Controlled via CSS selectors:
  ```css
  html.lenis {
    height: auto;
  }
  .lenis-smooth {
    scroll-behavior: auto;
  }
  ```

### 5.2 Micro Interactions & Reveals (Framer Motion)
- Enforced on lists, cards, and text reveals.
- Staggered entrances use declarative variants:
  ```typescript
  export const staggerContainer = {
    hidden: { opacity: 0 },
    show: {
      opacity: 1,
      transition: { staggerChildren: 0.15 }
    }
  };
  ```

### 5.3 Cinematic Timelines (GSAP)
- Restricted exclusively to the Hero block and custom scroll-triggered canvas layers.
- Enforces garbage collection on component unmount to prevent memory leaks:
  ```typescript
  useEffect(() => {
    const ctx = gsap.context(() => {
      // timelines go here
    });
    return () => ctx.revert(); // clean revert
  }, []);
  ```

## 6. Performance Targets & Code Rules

### 6.1 Lighthouse KPI
- **Performance:** 90+ (Requires optimized Next.js Images, font-swapping, and lightweight JS bundles).
- **Accessibility:** 95+ (Requires `aria-*` tags, keyboard navigation traps on modals, and distinct text/background contrast ratios).
- **SEO:** 100 (Valid JSON-LD schema, robot definitions, sitemap file path, clean metadata).

### 6.2 Code Quality & Constraints
- **Zero Three.js / R3F:** Prevent GPU throttling and high bundle sizes. Replace complex 3D scenes with vector grids, WebGL canvases, and canvas-frame pre-renders.
- **Hardware-Accelerated Animation:** Use `transform` properties (`translate3d`, `scale`, `rotate`) instead of modifying layout triggers (like `top`, `margin`, `height`).
- **Asset Optimization:** Enforce WebP/AVIF formats for rasterized graphics. Embed vectors directly as inline SVGs.
